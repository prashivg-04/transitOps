## ✨ Features

### Core Functionality
- **Driver Management**: Track drivers, their credentials, and performance metrics
- **Vehicle Management**: Complete vehicle fleet tracking and maintenance scheduling
- **Trip Management**: Log and monitor trips with real-time data
- **Fuel Management**: Track fuel consumption and optimize fuel efficiency
- **Expense Tracking**: Manage operational and maintenance expenses
- **Maintenance Scheduling**: Automated maintenance tracking and alerts
- **Advanced Analytics**: Comprehensive dashboards and reporting

### Security & Access Control
- JWT-based authentication
- OAuth2 integration support
- Role-based access control (RBAC)
- Secure password hashing
- Request logging and error handling

## 🛠️ Tech Stack

### Backend
- **Framework**: FastAPI (Python)
- **Database**: SQL-based (with Alembic migrations)
- **Authentication**: JWT + OAuth2
- **Server**: Uvicorn

### Frontend
- **Framework**: React with Vite
- **Styling**: Tailwind CSS
- **Build Tool**: Vite
- **Linting**: ESLint
- **CSS Processing**: PostCSS

## 🚀 Getting Started

### Prerequisites
- Python 3.8+ (Backend)
- Node.js 16+ (Frontend)
- Git

### Backend Setup

1. **Navigate to the backend directory**
   ```bash
   cd transitOps-backend
   ```

2. **Create a virtual environment**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Set up environment variables**
   Create a `.env` file in the `transitOps-backend` directory with required configuration:
   ```
   DATABASE_URL=your_database_url
   SECRET_KEY=your_secret_key
   ALGORITHM=HS256
   ACCESS_TOKEN_EXPIRE_MINUTES=30
   ```

5. **Run database migrations**
   ```bash
   alembic upgrade head
   ```

6. **Start the development server**
   ```bash
   uvicorn main:app --reload
   ```
   The API will be available at `http://localhost:8000`

### Frontend Setup

1. **Navigate to the frontend directory**
   ```bash
   cd transitOps-frontend
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up environment variables**
   Create a `.env` file in the `transitOps-frontend` directory:
   ```
   VITE_API_URL=http://localhost:8000/api
   ```

4. **Start the development server**
   ```bash
   npm run dev
   ```
   The application will be available at `http://localhost:5173`

5. **Build for production**
   ```bash
   npm run build
   ```

## 🏗️ Architecture

### Backend Architecture
- **Layered Architecture**: Separation of concerns with routers, services, CRUD, and models
- **Async Support**: Fully async Python implementation for high performance
- **Error Handling**: Centralized error handling with custom exceptions
- **Logging**: Request logging and error tracking middleware

### Frontend Architecture
- **Component-based**: Modular React components
- **API Integration**: Centralized API client for backend communication
- **RBAC Implementation**: Role-based access control at UI level
- **Responsive Design**: Mobile-friendly UI with Tailwind CSS

## 🔐 Security

- JWT tokens for API authentication
- Password hashing with industry-standard algorithms
- CORS protection
- Request validation with Pydantic
- Role-based access control (RBAC)
- Secure password storage and recovery

## 📖 Key Modules

### Backend Modules
- **auth**: Authentication, JWT, OAuth2, password management
- **crud**: Create, read, update, delete operations for all entities
- **services**: Business logic for drivers, vehicles, trips, expenses, etc.
- **models**: Database models for all entities
- **schemas**: Request/response validation schemas
- **routers**: REST API endpoints

### Frontend Components
- **Auth Components**: Login and authentication flows
- **Dashboard**: Main analytics and overview page
- **Management Pages**: Drivers, vehicles, trips, expenses, fuel logs, maintenance
- **Reports**: Analytics and reporting views

## 🔄 Workflow

1. Frontend sends requests to the FastAPI backend
2. Backend authenticates requests using JWT
3. CRUD operations query the database
4. Services handle business logic
5. Responses are validated against Pydantic schemas
6. Frontend displays data with React components
